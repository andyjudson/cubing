F2L intuitive cases

Like with the solving the cross, solving the first two layers, is a combination of a few repeated patterns, and intuitively setting up pieces to allow you to slot an edge-corner pair into the position. Whilst there are over 40 algorithms, there are 4 easy cases when pieces are in the top layer, and you can use a few tricks to get other cases of pairs setup ready to insert, for example if the corner is in slot but the edge is in the top layer. CubeHead explains this really well, way better than I ever could, so watch his video, but the gist of F2L breaks down as: 1. find a pair, 2. setup easy inserts (i.e. pairs in top layer + disconnected), 3. solve pair in correct slot

stage 1 :: easy pairs
righty pair
 U R U' R' side colours match to right, push pair aside, open slot, insert pair, close slot
lefty pair U' L' U L side colours match to left, push pair aside, open slot, insert pair, close slot
righty insert R U R' diff colours facing up, white to right, create pair, insert pair, realign cross
lefty insert L' U' L diff colours facing up, white to left, create pair, insert pair, realign cross
stage 2 :: complex pairs
edge in slot, corner in top layer (R U R')
... (easy-insert)		 hold edge on right, put the corner front left and do ...
corner in slot, edge in top layer (R U R')
... (easy-insert)		 hold edge on the left, do same moves ...
pair connected in slot, but not solved (R U' R')
... (easy-insert)		 hold pair on right, same but u-prime to disconnect...
pair connected in top layer, but not solved (R U2 R')
... (easy-insert)		 hold corner on right, above slot not yet solved, do ...
stage 3 :: turn pairs into easy inserts
  1. find the correct slot where pair needs to go.
  2. look at corner, where does white face?
If white faces to the side ... hold above corner slot, turn upper layer once, so can still see white, then check if the pair has matching top colour?
If pair match, create an easy pair case:
  1. turn slot up, putting corner to safety
  2. turn upper layer, putting edge where needs to be for pair
  3. realign the cross, bring corner back up
  4. solve pair with insert, ...
If pair do not match, create an easy insert case:
  1. turn slot up, putting corner to safety
  2. turn upper layer, putting edge where needs to be for pair
  3. realign the cross, bring corner back up
  4. solve pair with insert ...
If white faces up ...
  1. turn edge so over the correct centre of side face
  2. turn edge away, by bringing slot up
  3. turn corner above edge
  4. realign the cross
  5. solve pair with insert ...

OLL 2-look cases

The goal of OLL (orientation of last layer) is to solve the yellow face after finishing F2L. We can no longer use "intuitive" techniques, rather we need to use algorithms that solve specific patterns. Even though, there are 57 cases in total, we only need 9 algorithms for 2-look OLL. Most algorithms begin by taking one pair out (often front-right or front-left), doing something with it and then putting it back. To memorize the entire algorithm more easily, try following what is happening to that pair.

PLL 2-look cases

The goal of PLL (permutation of the last layer) is to finish the cube solve by moving pieces around after OLL. You do this by recognizing patterns on the side of the last layer, and do an algorithm that solves that case. Even though there 21 cases in total, we only need 6 algorithms for 2-look PLL. The first step is to solve the corners, then the second step is to solve the edges.
