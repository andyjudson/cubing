CFOP Beginner Method

The site is built to make it easier to practice (i.e. replay) the specific moves per case whilst learning CFOP algorithms. All examples and notes are based on the excellent Beginner CFOP tutorials from CubeHead.

I'll focus on the F2L, OLL and PLL stages, and will skip the CROSS stage but see the videos from CubeHead. Just to be clear, I'm not a cubing expert, far from it, this is a cheat sheet for my own learning, and if others find it useful, then that is good to hear.

Intuitive F2L 2-Look OLL 2-Look PLL

F2L intuitive cases

See Intuitive F2L YouTube video from CubeHead.

Like with the cross stage, solving the first two layers, is a combination of a few repeated patterns, and intuitively setting up pieces to allow you to slot an edge-corner pair into the position. Whilst there are over 40 algorithms, there are 4 easy cases when pieces are in the top layer, and you can use a few tricks to get other cases of pairs setup ready to insert, for example if the corner is in slot but the edge is in the top layer. CubeHead explains this really well, way better than I ever could, so watch his video, but the gist of F2L breaks down as: 1. find a pair, 2. setup easy inserts (i.e. pairs in top layer + disconnected), 3. solve pair in correct slot.

Step 1 :: Easy Inserts
1.1. Righty pair

U R U' R'

Side colours match to right. Push pair aside, open slot, insert pair, close slot.

Prob. ??
1.2. Lefty pair

U' L' U L

Side colours match to left. Push pair aside, open slot, insert pair, close slot.

Prob. ??
1.3 Righty insert

R U R'

Different colours facing up, white to right. Create pair, insert pair, realign cross.

Prob. ??
1.4 Lefty insert

L' U' L

Different colours facing up, white to left. Create pair, insert pair, realign cross

Prob. ??
Step 2 :: Fix unsolvable (stuck and/or connected) pairs
2.1. Edge stuck in slot, corner in top layer

R U R' ...

Hold edge on right, put the corner front left and use above algorithm (insert trigger) to bring pair into top layer and disconnected. Then setup or use an easy insert to solve.

Prob. ??
2.2. Corner stuck in slot, edge in top layer

R U R' ...

Hold edge on left, put the corner front right and use above algorithm (insert trigger) to bring pair into top layer and disconnected. Then setup or use an easy insert to solve.

Prob. ??
2.3. Pair stuck unaligned in slot

R U' R' ...

Hold pair on right, similar algorithm, but this time we use u-prime to bring the pair into top layer and disconnect them. Then setup or use an easy insert to solve.

Prob. ??
2.4 Pair stuck unaligned and connected in top layer

R U2 R' ...

Hold corner on right, above an unsolved slot, do algorithm. Then setup or use an easy insert to solve.

Prob. ??
Step 3 :: Solve pairs in target slot
3.1 White face to side + colours match

If white face is to the side, and colours match, you need to move the edge next to the corner, to setup an easy pair case.

  1. find the correct slot where corner needs to go
  2. turn upper layer once so can still see white
  3. turn target slot up, putting corner to safety
  4. turn upper layer, putting edge where needs to be for pair (next to)
  5. realign the cross, which brings corner back up
  6. solve pair with a letft or right easy insert, depending on which side matches
Prob. ??
3.2 White face to side + colours not match

If white face is to the side, and if colours match, you need to move the edge next to the corner, to setup an easy pair case.

  1. find the correct slot where corner needs to go
  2. turn upper layer once so can still see white
  3. turn slot up, putting corner to safety
  4. turn upper layer, putting edge where needs to be for pair (across from)
  5. realign the cross, bring corner back up
  6. solve pair with a letft or right easy insert, depending on which side matches
Prob. ??
3.3 White face up

If white face up ...

  1. turn edge to align with the matching centre of side face
  2. turn slot up, putting edge to safety
  3. turn upper layer, putting corner above edge
  4. realign the cross
  5. solve pair with insert
Prob. ??

OLL 2-Look Cases

See 2-look OLL YouTube video from CubeHead.

The goal of OLL (orientation of last layer) is to solve the yellow face after finishing F2L. We can no longer use "intuitive" techniques, rather we need to use algorithms that solve specific patterns. Even though, there are 57 cases in total, we only need 9 algorithms for 2-look OLL. Most algorithms begin by taking one pair out (often front-right or front-left), doing something with it and then putting it back. To memorize the entire algorithm more easily, try following what is happening to that pair.

Note: there are several funky named, triggers (denoted by round brackets) that you'll learn to use, for example: sexy (R U R' U') trigger features regularly in these algorithms, also common are: inverse sexy (U R U' R'), sledge (R' F R F').

Step 1 :: Cross Orientation
1.1. Line (horizontal)

F (R U R' U') F'

Prob. 1/4
1.2. Hook (front/right)

f (R U R' U') f'

Prob. 1/2
1.3. Dot

F (R U R' U') F' f (R U R' U') f'

Prob. 1/8
Step 2 :: Corner Orientation
2.1. Sune

(R U R' U) (R U2 R')

Hold the "fish" mouth front left so facing you, with a yellow side facing you on the front right

Prob. 4/27
2.2. Anti-Sune

(R U2 R') (U' R U' R')

If fish is facing you, but the yellow piece facing to the side, do U2 to rotate the top layer, moving a yellow piece to face you in the front left and the fish orientated back right, then do the algorithm.

Prob. 4/27
2.3. Cross-1

(R U R' U) (R U' R' U) (R U2 R')

This is first cross case, hold cube so the headlights face to the side.

Prob. 2/27
2.4. Cross-2

R U2 (R2 U') (R2 U') R2 U2 R

This is the second cross case, this time the headlights are more like side lights facing outwards, hold the cube so headlights on the left and side lights on the right.

Prob. 4/27
2.5. Chameleon

(r U R' U') (r' F R F')

Hold the block to the right, with too outward facing stickers to the left.

Prob. 4/27
2.6. Kite

F' (r U R' U') (r' F R)

Hold the cube so two corners, one top right, one bottom left. You should have a yellow piece facing you front right.

Prob. 4/27
2.7. Headlights

R2 D (R' U2 R) D' (R' U2 R')

Hold the yellow block, towards the back with headlights facing you.

Prob. 4/27

PLL 2-Look Cases

See 2-look PLL YouTube video from CubeHead.

The goal of PLL (permutation of the last layer) is to finish the cube solve by moving pieces around after OLL. You do this by recognizing patterns on the side of the last layer, and do an algorithm that solves that case. Even though there 21 cases in total, we only need 6 algorithms for 2-look PLL. The first step is to solve the corners, then the second step is to solve the edges.

Step 1 :: Corner Permutation
1.1. Opposite corner swap (Y perm)

F R U' R' U' R U R' F'
(R U R' U') (R' F R F')

If no headlights on any side, you need to swap diagonal corners.

Prob. 2/3
1.2. Adjacent corner swap (T perm)

(R U R' U') R' F
R2 U' R' U' R U R' F'

If have headlights, hold them on the left side, use the algorithm to swap adjacent corners.

Prob. 1/6
Step 2 :: Edge Permutation
2.1 Edge cycle anti-clockwise (Ua perm)

(R2 U' R') U' R
(U R) (U R) U' R

Hold the cube so the bar is facing you, the edge of left is opposite its corners.

Prob. 1/3
2.2 Edge cycle clockwise (Ub perm)

R' U (R' U') (R' U')
R' U (R U R2)

Hold the cube with the bar facing you, and the edge on the right opposite its corners.

Prob. 1/3
2.3 Opposite edge swap (H perm)

M2 U' (M2 U2 M2) U' M2

This algorithm uses middle layer moves. For this case, we have a checkerboard pattern all around the cube, each edge is opposite of the corners.

Prob. 1/12
2.4 Adjacent edge swap (Z perm)

M' U' (M2 U')
(M2 U') M' U2 M2

Another checkerboard pattern, but this time edges are not opposite the corners, orientate the top layer or cube so that the edges that need to swap are in the front and right slots.

Prob. 1/6
